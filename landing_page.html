FAQ

Questions? We Have Answers.

How is EuroScale different from PlanetScale?
EuroScale offers the same serverless database branching and horizontal scaling workflow as PlanetScale — but built exclusively on European cloud infrastructure (Hetzner, OVHcloud, Exoscale). All data stays within EU borders, making us GDPR-compliant by architecture. We also support both MySQL (via Vitess) and PostgreSQL (via Citus), whereas PlanetScale is MySQL-only.
Is my data protected under GDPR?
Absolutely. EuroScale is designed for GDPR compliance from the ground up. All data is stored exclusively on EU-based servers, never leaves EU/EEA borders, and is protected by strict data processing agreements (DPAs). We are Schrems II compliant, Gaia-X ready, and offer data residency guarantees in your choice of German, French, or Swiss data centers. Unlike US-based providers, your data is not subject to the US CLOUD Act.
What databases do you support?
EuroScale supports MySQL 8.0+ via Vitess for horizontal scaling (PlanetScale-compatible), and PostgreSQL 15+ via Citus for distributed Postgres workloads. Both engines support database branching, point-in-time recovery, and zero-downtime schema migrations. You can mix and match engines across your databases.
Can I use my own cloud infrastructure?
No. EuroScale runs on our own highly available Kubernetes cluster spanning Hetzner, OVHcloud, and Exoscale across multiple EU regions. You get a connection string — nothing else to provision. Enterprise customers can request dedicated clusters within the same EuroScale-managed multi-cloud infrastructure, but you never manage servers, Kubernetes, or cloud accounts yourself.
How does database branching work?
Database branching creates an instant, isolated copy of your production database for development, testing, or CI/CD. Under the hood, EuroScale uses Vitess/VReplication to clone the schema and establish a copy-on-write data layer — no full data copy is needed, so branches are created in seconds regardless of database size. When you're ready, deploy the branch back to production with zero downtime.
What happens if one of my cloud providers goes down?
EuroScale's multi-cloud architecture means you can run database replicas across different European providers simultaneously. With the Business and Enterprise plans, we support cross-cloud failover — if Hetzner has an outage in Nuremberg, your traffic automatically routes to replicas on OVHcloud or Exoscale. Combined with our 3-node HA clusters, we target 99.99% uptime for Enterprise customers.
Are there any hidden egress fees?
No. EuroScale has zero hidden egress fees. Unlike AWS (which charges €0.05–0.09/GB for data transfer) and other US hyperscalers, we believe data transfer should be free. Your pricing includes all inbound and outbound data transfer within the EU region. This is part of our commitment to transparent European pricing.
How do I migrate from PlanetScale or AWS RDS?
EuroScale provides a migration CLI tool that handles schema translation and data replication with minimal downtime. For PlanetScale users, the Vitess-based architecture means your existing MySQL branching workflow ports directly. For AWS RDS, we support live replication via binlog (MySQL) or logical replication (PostgreSQL). Most migrations complete in under an hour. Our support team provides white-glove migration assistance for Business and Enterprise plans.
