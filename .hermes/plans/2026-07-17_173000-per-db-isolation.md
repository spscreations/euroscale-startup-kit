# Per-DB Isolation — Enterprise Tier Architecture

## Design
Each database gets a dedicated vtgate cell with unique port, hostname, TLS, and network isolation.

**Path:** Client → `fff.db.euroscale.app` → LB port → Traefik SNI → dedicated vtgate → own keyspace tablets

## Isolation Layers

| Layer | Mechanism |
|---|---|
| Network | CiliumNetworkPolicy: ingress only from Traefik pods + same-DB vttablets |
| Port | Dedicated vtgate port (3306+n, n=DB index) |
| Hostname | `{name}.db.euroscale.app` A record → LB IP |
| Authentication | mTLS: per-DB client cert signed by per-DB CA |
| Keyspace | Database-specific keyspace in shared VitessCluster |

## Implementation

### 1. VitessCluster: add per-DB cell
Patch `spec.cells` to add a new cell per database:
```yaml
- name: db-fff
  gateway:
    replicas: 1
    extraFlags:
      mysql_server_port: 3307  # unique per DB
      ...
    secureTransport:
      required: true
      tls:
        certSecret: {name: "db-fff-vtgate-tls", key: "tls.crt"}
        keySecret: {name: "db-fff-vtgate-tls", key: "tls.key"}
        clientCACertSecret: {name: "db-fff-ca", key: "ca.crt"}
```

### 2. cert-manager: per-DB CA + vtgate TLS
- Self-signed issuer → per-DB CA cert
- CA issuer → vtgate server cert with SAN `{name}.db.euroscale.app`
- Client cert for user signed by per-DB CA

### 3. Traefik: SNI TCP route
```yaml
apiVersion: traefik.io/v1alpha1
kind: IngressRouteTCP
metadata:
  name: db-fff
  namespace: euroscale
spec:
  entryPoints: [mysql]
  routes:
  - match: HostSNI(`fff.db.euroscale.app`)
    services:
    - name: euroscale-db-fff-vtgate  # service auto-created by vitess-operator
      port: 3307
```

### 4. CiliumNetworkPolicy: per-DB isolation
```yaml
apiVersion: cilium.io/v2
kind: CiliumNetworkPolicy
metadata:
  name: isolate-db-fff
  namespace: euroscale
spec:
  endpointSelector:
    matchLabels:
      planetscale.com/cell: db-fff
  ingress:
  - fromEndpoints:
    - matchLabels:
        app.kubernetes.io/name: traefik
    - matchLabels:
        planetscale.com/keyspace: fff
```

### 5. DNS: `{name}.db.euroscale.app` A → LB IP
- Wildcard or per-record: `*.db.euroscale.app` → 142.132.242.220
- OR per-DB A record via Cloudflare API

### 6. API: CreateDatabase provisions all of the above
- Generate cell name (`db-{name}`)
- Provision cert-manager CA + vtgate cert
- Patch VitessCluster with new cell
- Create IngressRouteTCP
- Create CiliumNetworkPolicy
- Generate client cert
- Return hostname + port + certs

### 7. API: DeleteDatabase tears down all of the above
- Remove cell from VitessCluster
- Delete cert-manager certs
- Delete IngressRouteTCP
- Delete NetworkPolicy
- Clean up secrets

## Resource cost per DB (enterprise)
| Resource | RAM | CPU |
|---|---|---|

| vttablet (existing, shared) | ~0 MB (shared pool) | ~0 (shared) |
| certs | 0 MB | 0 |
| **Total** | **~130 MB** | **~0.1 CPU** |
